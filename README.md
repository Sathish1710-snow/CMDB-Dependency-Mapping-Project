# Project 7 - CMDB-Dependency-Mapping-Project
Designed a CMDB Dependency Mapping solution using Business Services, CI relationships, and Business Rules to automate service mapping and improve incident impact analysis.

I recently built a CMDB Dependency Mapping project to understand how business services are connected with underlying infrastructure and how failures impact the overall system.

Project Highlights:  Created Business Service – Banking Service
- Designed and managed multiple Configuration Items (Web, App, DB Servers)
- Established CI relationships to model real-time dependencies
- Implemented Dependency View for impact analysis
- Tested scenarios by raising incidents on affected CIs
- Automated business service population using Business Rules

Key Outcome: 

When a critical CI (like Database Server) goes down, the system automatically identifies impacted services, enabling faster root cause analysis and better decision-making.

This project helped me strengthen my understanding of:
- CMDB structure & CI lifecycl
- Service Mapping & Dependency relationships
- Incident impact analysis
- Auto populate the Service
